## MCP Connection
- **Port**: {port}
- **Your Agent ID**: {agent_id}
- **Your Lead ID**: {lead_id}

### Send message
CRITICAL: Do NOT use curl — it breaks Unicode on Windows. Use PowerShell instead.
Set `recipient_id` to `{lead_id}` (your lead) or another agent's ID. Replace YOUR_MESSAGE with your text.
```powershell
powershell -NoProfile -Command "$b = @{sender_id='{agent_id}'; recipient_id='{lead_id}'; text='YOUR_MESSAGE'} | ConvertTo-Json; Invoke-RestMethod -Uri 'http://127.0.0.1:{port}/messages/send' -Method POST -ContentType 'application/json; charset=utf-8' -Body ([System.Text.Encoding]::UTF8.GetBytes($b))"
```

### Check your messages
```powershell
powershell -NoProfile -Command "Invoke-RestMethod -Uri 'http://127.0.0.1:{port}/messages/{agent_id}'"
```

### Discover other agents
`curl -s http://127.0.0.1:{port}/agents` — returns `[{"id","name","agent_type","task"}]`. Use `id` as `recipient_id`.

### Completion report (MANDATORY)
Before finishing (AGENT_COMPLETED), send a brief report to `{lead_id}` (what was done + problems, under 500 chars). No exceptions.

### Boundaries
You CAN: send/check messages, discover agents. You CANNOT: use MCP tools (`mcp__ShikigamiMCP__*`), read other agents' messages, register/unregister agents.
PREFER `USER_INPUT_REQUIRED` over messaging for blocking questions — the runner handles re-launch with the answer automatically.
